Mis proyectos

Hechos con VueJs

De los proyectos creados con el Framework de VueJs se puede ver el proyecto de GuitarLA, el cual tiene funcionamiento de un carrito de compras agregando diferentes tipos de guitarras con sus respectivos precios, haciendo una suma total del precio a pagar y con la opcion de vaciar el carrito.

El segundo proyecto es un Cotizador de Criptomonedas, el cual se conecta a una api para obtener el precio de alguna criptomoneda en algún tipo de moneda como pesos mexicanos o dólar.

El tercer proyecto es un Control de Gastos en donde se le establece un presupuesto inicial, donde una vez agregado pasa a otra pantalla en la que se pueden visualizar los gastos que has hecho partiendo del mismo presupuesto inicial. Por lo que si no se ha añadido ningun gasto aparecerá como "sin gastos", y para añadir nuevos gastos se debe dar clic en el icono de agregar posicionado en la parte inferior derecha identificado por un "+".

Esto abre un modal donde se le pone un nombre al gasto que está por hacerse, así como la cantidad del gasto y seleccionando una categoría perteneciente al tipo de gasto.

El cuarto proyecto es un Administrados de Pacientes de una veterinaria en donde se debe llenar un formulario con el respectivo nombre de la mascota, el nombre del propietario, así como su email, una fecha de alta y los sintomas que tenga dicha mascota para así ser atendida.

El quinto proyecto es un Buscador de Clima en donde se debe escribir el nombre de alguna ciudad así como su país, estos dos campos deben coincidir puesto que este proyecto se conecta a una api en la cual busca el clima de dicha ciudad proporcionada con anterioridad y coincidente con el país escogido.

El sexto proyecto es un Buscador de bebidas el cual accede a una API que contiene todas las bebidas, por lo que el sistema necesita que se llenen ambos campos de categoria y nombre o ingrediente que deseas buscar. Y así poder ir marcando tus bebidas favoritas y acceder a ellas por medio de la pestaña de favoritos, guardando los datos en local storage.

El séptimo proyecto es una página estilo Bienes Raíces en la cual se visualiza de primera instancia las casas en venta, pero si se intenta ingresar a la sesión usando el correo "correo@correo.com" y la contraseña "123456" se toma el usuario con el que agregaron las casas, pudiendo ir a la sección de administracion, y poder editar la información de las casas así como eliminarlas. Para este proyecto se hizo uso de Firebase.

Nota: Debido al uso de Firebase debo volver configurarlo cada mes, por lo que si no puede acceder correctamente al sitio, puede checarlo en otro momento.

El octavo proyecto es un Punto de Venta donde de primera instancia se aprecia un filtro en funcionamiento, pudiendo clasificarse por sudaderas, tenis y lentes. Además se pueden agregar productos a un carrito de compras, un ejemplo pueden ser las cajas que usan en las tiendas deportivas para vender sus artículos, concluyendo con registrar una compra, incluyendo o no cupones de descuento. Para después ir al panel de administración y encontrar un calendario, donde te aparecen los datos o ventas totales de cierto día, junto con toda la información relevante para el vendedor. Pero aparte en el panel de administración hay una opción más llamada productos, donde puedes agregar, eliminar o editar los productos, como actualizarlos cuando llega nueva mercancía, ya que cuenta con numeros de existencia por producto. Para este proyecto se hizo uso de Firebase.

Nota: Debido al uso de Firebase debo volver configurarlo cada mes, por lo que si no puede acceder correctamente al sitio, puede checarlo en otro momento.

Hechos con PHP MVC SASS POO MySQL

El primer proyecto hecho con PHP MVC es un sistema de Bienes Raíces, el cual incluye una página principal en donde se muestran muchos datos acerca de alguna empresa, aparecen 3 anuncios con disponibilidad para ver independientemente los aspectos de cada una de las propiedades, y algunas entradas de blog.

Cuenta con una barra de navegación con un apartado de "Nosotros", "Anuncios", "Blog", y "Contacto", todo comunicado de forma dinámica e incluyendo un modo oscuro. El apartado de "Contacto" tiene un formulario con el que se puede escoger el método de contacto, siendo email o telefono y ofreciendo diferentes campos a llenar dependiendo del método que se escogió.

Por otra parte tiene un login al cual se debe acceder con la URL https://nilwita.nyc.dom.my.id/login, se debe poner el correo de acceso "correo@correo.com" y usando la contraseña "123456", con lo que se obtiene el acceso a una ruta protegida de administrador, el cual puede ver, agregar, editar o eliminar propiedades, así como más vendedores. Todo interactuando dinámicamente con la página principal que puede ver cualquier usuario.

El segundo proyecto de PHP MVC es un AppSalon, es decir una aplicación para agendar citas, en este caso una barbería o salón de belleza, el cual cuenta con un registro de usuarios que tienen que ser confirmados por medio de correo electronico, y ya que está programado con Mailtrap no es posible hacerlo con correos reales y por esta misma razón no es posible crear cuentas, ha no ser que se tenga acceso a la base de datos y se escriba el token asignado en la URL de confirmación de cuenta, o también contactandome para pasarles el link de confirmación.

Una vez inicies sesión con el correo "correo@correo.com" y la contraseña "password" accedes a la ruta protegida para poder agendar citas, para las citas se debe escoger al menos 1 servicio, y pasar a la siguiente seccion donde se escoge la fecha (no debe ser fin de semana) y hora (entre 10am y 6pm disponibles), para finalizar en la última sección donde se confirma la cita mientras se ve un resumen de la misma.

Una vez agregada la cita en dicha fecha puede cerrar sesión o agregar tantas citas como quiera y después pasar al area de administración. Accediendo ahora con el correo de "admin@admin.com" y contraseña "12345678", teniendo de primera vista las citas del día siguiente al actual en que se este visitando el sitio, o la fecha que se escoja. Por otra parte se pueden ver, agregar, editar, o eliminar servicios del salón/barbería.

En el tercer proyecto de PHP MVC es un Administrador de Tareas el cual funciona con un registro de usuarios que tienen que ser confirmados por medio de correo electronico, y ya que también está programado con Mailtrap no es posible hacerlo con correos reales y por esta misma razón no es posible crear cuentas, ha no ser que se tenga acceso a la base de datos y se escriba el token asignado en la URL de confirmación de cuenta, o también contactandome para pasarles el link de confirmación.

Una vez que se inicie sesión con el correo "correo@correo.com" y la contraseña "123456" se accede al panel principal del administrador de tareas, donde se visualian los proyectos creados, en caso de no tenerlos se pueden crear desde la barra lateral, y una vez creado un proyecto se puede acceder a el, y empezar a añadir tareas, así como visualizarlas todas, editarlas dando doble click sobre la tarea, cambiar el estado de la tarea de pendiente a completada (dando doble click sobre el botón del estado de la tarea), o eliminandola (de igual forma dando doble click en eliminar). Y en caso de querer modificar los datos del perfil, se accede al apartado de perfil y cambias el correo o la contraseña sin necesidad de enviar un correo ya que se accedió con las credenciales que dieron a conocer anteriormente.

Nota: También incluye un modo oscuro que toma la elección que tiene el usuario desde su sistema
